# CCF-A 论文深度分析：开源工具调研报告

> 调研日期：2026-03-03
> 目的：梳理可用于 `data/paper/conf_a` 深度分析的成熟开源库，避免重复造轮子

---

## 一、背景与问题诊断

### 现有代码的核心局限

当前 `analysis/` 目录的实现存在以下方法论缺口：

| 模块 | 现有实现 | 核心问题 |
|---|---|---|
| `trend_analysis.py` | 手动分词 + 单词级词频统计 | **无法捕捉多词短语**，"large language model" 被拆成三个无意义词 |
| `deep_domain.py` | 硬编码关键词表 + word boundary 匹配 | 仅 surface-level 匹配，无语义理解；增长率无统计显著性 |
| `topic_model.py` | TF-IDF + KMeans/DBSCAN | 已引入 SentenceTransformer 但为可选依赖，实际退化为 TF-IDF |
| `trend_analysis.py` | `analyze_author_trends` 仅统计论文数 | **无合作者网络**，无中心性分析，无社区发现 |
| 全局 | 无 N-gram PMI 分析 | 技术词组（"graph neural network"、"federated learning"）无法被识别为整体 |

### 分析目标

基于 13 个 CCF-A 顶会（2015–2025）约 20 万篇论文，实现以下深度分析：

1. **研究生态宏观演化** — 范式跃迁的量化定位
2. **技术概念生命周期** — S 曲线拟合，词汇半衰期
3. **N-gram 概念图谱** — PMI 驱动的多词短语提取，概念共现网络
4. **会议定位与竞争格局** — 各顶会的研究重心差异化分析
5. **作者合作网络** — 跨领域桥接者、研究社区边界
6. **语义相似度分析** — 基于 SPECTER2 的论文聚类

## 二、工具调研总览

### 2.1 主题建模

#### BERTopic
- **GitHub**：[MaartenGr/BERTopic](https://github.com/MaartenGr/BERTopic)
- **Stars**：~7,200
- **维护状态**：活跃（最新版本 2025 年 12 月）
- **核心功能**：基于 Transformer 嵌入 + UMAP + HDBSCAN + c-TF-IDF 的神经主题建模
- **对本项目的价值**：
  - 替代 `TopicModeler` 中的 TF-IDF + KMeans 管道，产出语义一致的主题
  - **`topics_over_time` 方法**直接实现主题随年份的演化追踪，可替代 `analyze_keyword_trends` 中的手工时序逻辑
  - 内置可视化：主题地图、时序热图、主题层次树
- **关键能力**：
  ```
  BERTopic.fit_transform(abstracts)        → 自动发现主题及关键词
  BERTopic.topics_over_time(texts, years)  → 主题的年度演化曲线
  BERTopic.find_topics("federated learning") → 语义搜索相关主题
  BERTopic.get_topic_info()               → 主题统计汇总
  ```
- **注意事项**：处理 20 万篇论文需要 GPU 或预计算嵌入；建议分批次或使用 `OnlineBERTopic`

### 2.2 关键词与短语提取

#### KeyBERT
- **GitHub**：[MaartenGr/KeyBERT](https://github.com/MaartenGr/KeyBERT)
- **Stars**：~4,100
- **维护状态**：活跃（2025 年 2 月）
- **核心功能**：基于 BERT 嵌入和余弦相似度提取关键短语，原生支持 N-gram（1–3 词）
- **对本项目的价值**：替代 `trend_analysis.py` 和 `deep_domain.py` 中分散的 `words = text.lower().split()` 逻辑，直接产出如 "graph neural network"、"retrieval augmented generation" 这样的完整技术词组
- **关键参数**：`keyphrase_ngram_range=(1,3)`、`use_maxsum=True`（多样性去重）、`nr_candidates=20`

#### YAKE（Yet Another Keyword Extractor）
- **GitHub**：[LIAAD/yake](https://github.com/LIAAD/yake)
- **Stars**：~1,800
- **维护状态**：活跃
- **核心功能**：无监督、统计型、语言无关的关键词提取，无需模型下载，速度比 KeyBERT 快 10–100 倍
- **对本项目的价值**：大批量处理（20 万摘要）时的首选；无 GPU 依赖
- **适用场景**：快速批处理整个 conf_a 数据集；作为 KeyBERT 的轻量替代

#### RAKE-NLTK
- **GitHub**：[csurfer/rake-nltk](https://github.com/csurfer/rake-nltk)
- **Stars**：~992
- **维护状态**：基本停止更新（2019）
- **适用场景**：基线对比；无依赖的快速原型

**三者比较**：

| 工具 | 速度 | 质量 | 多词支持 | GPU 需求 | 推荐场景 |
|---|---|---|---|---|---|
| YAKE | 极快 | 中 | ✓ | 无 | 大批量预处理 |
| KeyBERT | 中 | 高 | ✓ | 可选 | 代表性论文精析 |
| RAKE | 快 | 低 | ✓ | 无 | 基线对比 |

### 2.7 N-gram / PMI 短语提取

#### Gensim Phrases（推荐）
- **GitHub**：[piskvorky/gensim](https://github.com/piskvorky/gensim)
- **Stars**：~16,200
- **维护状态**：活跃
- **核心功能**：`gensim.models.phrases.Phrases` 使用 NPMI（归一化点互信息）检测统计显著的多词短语；支持链式叠加构建 bigram → trigram
- **对本项目的价值**：
  - **最高优先级的改进点**：当前所有关键词分析均为单词级别，Gensim Phrases 将整个语料库中的"large_language_model"、"graph_neural_network"作为完整 token 处理
  - 链式处理：先 bigram 再 trigram，可发现三词技术术语
  - `FrozenPhrases` 适合批量推理
- **参数调优**：`min_count=5`（最低出现次数）、`threshold=10`、`scoring="npmi"`

#### NLTK Collocations（探索性分析）
- **GitHub**：[nltk/nltk](https://github.com/nltk/nltk)（已被 NLTK 内置）
- **Stars**：~13,900
- **核心功能**：`BigramCollocationFinder` + PMI/卡方/似然比等多种显著性检验
- **与 Gensim 对比**：NLTK 更适合**统计探索**（查看不同 PMI 阈值的效果）；Gensim Phrases 更适合**管道化处理**（整个语料库的一致转换）
- **推荐用途**：确定最优 PMI 阈值，再用 Gensim 构建生产管道

### 2.8 文本时序分析与趋势统计检验

#### pymannkendall
- **GitHub**：[mmhs013/pymannkendall](https://github.com/mmhs013/pymannkendall)
- **Stars**：~400
- **维护状态**：活跃
- **核心功能**：Mann-Kendall 非参数趋势检验，专门用于检测时序数据中的单调趋势，返回**统计显著性 p 值**和 Theil-Sen 斜率估计
- **对本项目的价值**：
  - 直接解决现有代码最大的方法论缺陷 — `_analyze_emerging_topics` 中 `(late - early) / early > 0.5` 的判断没有统计意义
  - 对每个关键词的年度频率序列运行 Mann-Kendall 检验，只报告 p < 0.05 的**显著趋势**，消除小样本假阳性
  - `mk.original_test(keyword_freq_list)` 返回：趋势方向、p 值、Kendall's tau、Sen's 斜率

#### statsmodels
- **GitHub**：[statsmodels/statsmodels](https://github.com/statsmodels/statsmodels)
- **Stars**：~10,000
- **维护状态**：活跃
- **核心功能**：完整时序分析工具箱（ARIMA、STL 分解、平稳性检验）
- **对本项目的价值**：
  - `tsa.seasonal.seasonal_decompose`：将年度论文数趋势分解为趋势项 + 季节项 + 残差
  - `tsa.stattools.adfuller`：单词平稳性检验，区分真实增长趋势与随机波动

#### Arabica
- **GitHub**：[PetrKorab/Arabica](https://github.com/PetrKorab/Arabica)
- **Stars**：~76
- **维护状态**：活跃（2024 年发表于 JOSS）
- **核心功能**：专为文本数据的时序 N-gram 频率分析设计；按月/年聚合 unigram/bigram/trigram 频率；内置热图和折线图
- **对本项目的价值**：
  - 直接替代 `TrendAnalyzer.analyze_keyword_trends` 中的 `keyword_by_year` 字典构建逻辑
  - 输出即可可视化的 DataFrame，按年统计关键词/短语频率
  - 内置停用词过滤和时序平滑

#### BERTopic `topics_over_time`（功能更强）
- 已在 2.1 节描述；是时序文本分析的最强方案
- 与 Arabica 的区别：Arabica 追踪**预定义词汇**的频率；BERTopic 自动发现**潜在主题**并追踪其演化

## 三、优先级矩阵

按**影响力 / 实现成本**排序：

| 优先级 | 工具 | 替代的现有逻辑 | 预期提升 |
|---|---|---|---|
| ★★★★★ | **Gensim Phrases** | `words = text.lower().split()` 的所有用处 | 关键词提取质量根本性改善 |
| ★★★★★ | **BERTopic** | `TopicModeler` + `analyze_keyword_trends` 时序部分 | 语义主题 + 时序演化一体化 |
| ★★★★☆ | **YAKE / KeyBERT** | `_extract_keywords` 方法 | 多词关键短语提取，告别单词碎片 |
| ★★★★☆ | **NetworkX** | `analyze_author_trends`（目前仅计数） | 共作者图、中心性、社区发现 |
| ★★☆☆☆ | **PyAlex** | 数据补充层（暂不接入） | 为每篇论文添加引用数、h-index（可选后续补充） |
| ★★★☆☆ | **sentence-transformers + allenai-specter** | `TopicModeler` 中 TF-IDF 向量化 | 语义感知聚类（代码已引入，未激活） |
| ★★☆☆☆ | **scispaCy** | 无现有实现（预处理层） | 缩写规范化、科学 NER |
| ★★☆☆☆ | **Arabica** | `keyword_by_year` 字典构建 | 时序 N-gram 可视化现成方案 |
| ★★☆☆☆ | **SPECTER2** | SPECTER 的升级版（HuggingFace） | 更高精度的科学论文嵌入 |
| ★★☆☆☆ | **pymannkendall** | `_analyze_emerging_topics` 中无显著性的增长率判断 | Mann-Kendall 非参数趋势检验，p 值过滤假阳性 |
| ★★☆☆☆ | **cdlib** | `analyze_author_trends` 无社区结构 | Leiden 算法划分研究社区 |
| ★★☆☆☆ | **scattertext** | 无现有实现（会议差异可视化） | 交互式多会议特征词对比图 |
| ★☆☆☆☆ | **pybliometrics** | 无（需 Scopus 付费授权） | 引用数据（可用 PyAlex 免费替代） |

## 四、架构建议

### 推荐分析管道

```
原始 JSON 数据（conf_a/*.json）
        ↓
[数据加载] data_loader.py（现有）
        ↓
[文本预处理] scispaCy 缩写检测 + 清洗
        ↓
[N-gram 提取] Gensim Phrases（bigram → trigram）
        ↓
         ├──→ [主题建模] BERTopic（发现主题 + topics_over_time）
         ├──→ [关键词提取] YAKE（批量）/ KeyBERT（代表性论文）
         ├──→ [语义嵌入] SPECTER2（降维后用于聚类和相似度）
         └──→ [数据补充] PyAlex（引用数 + h-index）
                        ↓
[网络分析] NetworkX（共作者图 + 社区发现）
        ↓
[可视化] PyVis（网络）+ BERTopic 内置（主题图）+ Arabica（时序）
        ↓
[输出] JSON 报告 + HTML 交互可视化
```

### 与现有代码的整合方式

- `preprocessing.py`：在 `EmbeddingGenerator` 中将 TF-IDF 替换为 SPECTER2；在 `TextPreprocessor` 中集成 scispaCy 缩写检测
- `topic_model.py`：`TopicModeler.fit_transform()` 替换为 BERTopic；`DomainClassifier` 保留作为验证层
- `trend_analysis.py`：`analyze_keyword_trends` 的核心词频逻辑替换为 Gensim Phrases + Arabica
- `deep_domain.py`：`_analyze_emerging_topics` 的增长率逻辑替换为 BERTopic `topics_over_time`
- 新增 `network_analysis.py`：基于 NetworkX 构建共作者图，实现 `analyze_author_trends` 的图化升级

### 2.10 可视化（补充）

#### scattertext
- **GitHub**：[JasonKessler/scattertext](https://github.com/JasonKessler/scattertext)
- **Stars**：~2,200
- **维护状态**：活跃
- **核心功能**：可视化不同文档类别中词汇的差异分布，生成交互式 HTML；横轴为一类文档的词频，纵轴为另一类，右上角是两者共有的高频词，右下角是前者特有词
- **对本项目的价值**：
  - 比较不同**会议**的特征词（CVPR vs. ACL vs. AAAI）— 直接可视化会议研究重心差异
  - 比较不同**年代**的特征词（2015–2018 vs. 2023–2025）— 可视化研究范式演变

#### pyLDAvis
- **GitHub**：[bmabey/pyLDAvis](https://github.com/bmabey/pyLDAvis)
- **Stars**：~2,000
- **维护状态**：活跃
- **核心功能**：LDA 主题模型的交互式可视化，展示主题间距离和主题-词汇关系
- **对本项目的价值**：与现有 `topic_model.py` 中 scikit-learn LDA 管道直接对接

#### FAISS
- **GitHub**：[facebookresearch/faiss](https://github.com/facebookresearch/faiss)
- **Stars**：~31,000
- **维护状态**：活跃（Meta AI）
- **核心功能**：大规模稠密向量相似度搜索；处理百万级向量仅需毫秒
- **对本项目的价值**：为 20 万篇论文的 SPECTER2 嵌入构建**语义检索索引**，实现"找与某篇论文最相似的论文"等查询
