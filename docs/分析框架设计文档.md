## 三、方法论基础层

### 3.1 `analysis/ngram_extractor.py` — N-gram 短语提取

#### 解决的核心问题

当前所有模块均使用 `text.lower().split()` 进行单词级分词，导致"large language model"被拆成三个独立词，失去语义完整性。N-gram 提取是后续所有分析质量的基础。

#### 核心类：`NgramExtractor`

**初始化参数**：

| 参数 | 默认值 | 说明 |
|---|---|---|
| `min_count` | 5 | 短语最低出现次数 |
| `threshold` | 10 | NPMI 显著性阈值（越高越保守） |
| `scoring` | `'npmi'` | 评分方式，NPMI 归一化后更稳定 |
| `cache_dir` | `analysis/cache/` | 训练好的模型缓存路径 |

**主要方法**：

| 方法 | 输入 | 输出 | 说明 |
|---|---|---|---|
| `fit(texts)` | 文本列表 | `self` | 训练 bigram → trigram 两级短语模型 |
| `transform(text)` | 单条文本 | `List[str]` | 将文本转为 phrase-token 列表 |
| `fit_transform(texts)` | 文本列表 | `List[List[str]]` | 训练并批量转换 |
| `get_top_phrases(texts, top_n)` | 文本列表 | `List[Tuple[str, int]]` | 统计语料中最高频的 N-gram 短语 |

**示例效果**：

```
输入："We propose a graph neural network for federated learning"
输出：["propose", "graph_neural_network", "federated_learning"]

输入："large language model instruction tuning with RLHF"
输出：["large_language_model", "instruction_tuning", "rlhf"]
```

**技术实现要点**：
- 使用 `gensim.models.phrases.Phrases` + `Phraser` 两级串联（先 bigram，再 trigram）
- 训练前对文本做基本清洗：去标点、小写、过滤停用词
- 训练结果以 pickle 格式缓存到 `analysis/cache/ngram_model_{hash16}.pkl`，避免重复计算

### 3.2 `analysis/stats_utils.py` — 统计工具库

#### 解决的核心问题

现有 `_analyze_emerging_topics` 使用 `(late - early) / early > 0.5` 判断新兴话题，无统计意义。5 篇增至 10 篇（+100%）与 500 篇增至 1000 篇（+100%）被同等对待。

#### 主要函数

**`mann_kendall_test(series: List[float]) -> dict`**

对年度频率序列进行 Mann-Kendall 非参数趋势检验。

返回值：

```python
{
    "trend": "increasing" | "decreasing" | "no trend",
    "p_value": 0.023,      # 显著性，< 0.05 表示趋势显著
    "tau": 0.78,           # Kendall's tau，-1 到 1
    "slope": 12.4,         # Sen's 斜率（每年增长量）
    "significant": True    # p < 0.05
}
```

**`normalize_yearly(counts, totals) -> Dict[int, float]`**

将年度词频归一化为该会议该年论文总数的比例，消除会议规模差异（NeurIPS 每年 3500 篇 vs SIGMOD 每年 200 篇）。

**`compute_coverage_stats(papers) -> Dict`**

统计每年摘要覆盖率，为置信度标注提供依据。

**`format_insight(observation, comparison, implication, confidence) -> str`**

标准化洞见输出格式：

```
观察：LLM 论文从 2022 到 2024 增长了 340%
对比：ACL（+580%）比 CVPR（+120%）高出 5 倍（已归一化）
含义：当前 LLM 浪潮本质上仍是语言现象，视觉社区处于"翻译期"
置信度：基于 15,234 篇论文，摘要覆盖率 99.1%
```

## 四、宏观生态层

### 4.1 `analysis/ecosystem.py`

#### 模块 A：`VocabularyTimeline` — 词汇时间轴

**目标**：构建可量化的 AI 研究史料，精确定位每个重要技术短语从"首次出现"到"全面扩散"的时间节点。

**输出 DataFrame 列结构**：

| 列名 | 类型 | 说明 |
|---|---|---|
| `phrase` | str | N-gram 短语（如 "large_language_model"） |
| `first_year` | int | 首次在任意会议出现的年份 |
| `first_cluster_year` | int | 首次集中出现（≥10篇/年）的年份 |
| `peak_year` | int | 历史最高频年份 |
| `spread_speed` | float | 从 10篇/年 到 100篇/年 所需年数（NaN 表示未达到 100篇） |
| `trajectory` | str | "rising" / "peaked" / "declining" / "niche" |
| `origin_conf` | str | 最早集中出现的会议 |

**`get_paradigm_shifts(top_n=20)`** 按 `spread_speed` 升序排列，返回扩散最快的技术词汇——即真正的范式跃迁标志词。

**可回答的问题**：
- "self_attention" 何时从 ACL 扩散到 CVPR？花了几年？
- "chain_of_thought" 和 "instruction_tuning" 哪个扩散更快？
- 哪些词汇首次出现即成主流（扩散速度 < 1 年）？哪些长期保持小众？

---

#### 模块 B：`ConferenceSimilarityMatrix` — 会议相似度矩阵

**目标**：量化 LLM 时代前后 13 个顶会研究重心是否在融合，找出边界模糊最快的会议对。

**方法**：
1. 对每个会议的全部论文（标题+摘要）计算 TF-IDF 向量作为"会议指纹"
2. 计算 13×13 余弦相似度矩阵
3. 在三个时间窗口分别计算：2015–2018 / 2019–2022 / 2023–2025
4. 差值矩阵 = 2023–2025 矩阵 − 2015–2018 矩阵

**`compare_eras()` 输出**：

```python
{
    "era_2015_2018": np.ndarray,   # 13×13 相似度矩阵
    "era_2019_2022": np.ndarray,
    "era_2023_2025": np.ndarray,
    "delta": np.ndarray,           # 差值矩阵，正值=趋同，负值=分化
    "conf_names": List[str]        # 矩阵行/列对应的会议名
}
```

**可验证假设**：
- CVPR 与 ACL 的相似度在 2022 年后上升（多模态融合证据）
- SIGMOD/ICDE 与 ICML/NeurIPS 距离缩小（LLM for Databases 趋势）
- ICLR 与 NeurIPS 形成差异化定位（ICLR 偏方法，NeurIPS 偏理论）

---

#### 模块 C：`TechnologyDiffusion` — 技术扩散路径

**目标**：验证"专业会议首发 → 综合 AI/ML 扩散 → 应用领域渗透"的传播规律，量化不同技术的普适性差异。

**核心指标：扩散延迟**

> 定义：技术在发起会议首次达到 10 篇/年后，到目标会议首次达到 5 篇/年所需年数。

**`analyze_term(term, papers_by_conf, threshold=10)` 输出**：

```python
{
    "term": "transformer",
    "origin_conf": "ACL",
    "origin_year": 2017,
    "diffusion_path": [
        {"conf": "ACL",     "year": 2017, "delay": 0},
        {"conf": "NeurIPS", "year": 2018, "delay": 1},
        {"conf": "ICML",    "year": 2018, "delay": 1},
        {"conf": "CVPR",    "year": 2019, "delay": 2},
        {"conf": "SIGIR",   "year": 2019, "delay": 2},
        {"conf": "KDD",     "year": 2020, "delay": 3}
    ]
}
```

**可验证案例**：
- Transformer：ACL 2017 → NeurIPS/ICML → CVPR → SIGIR → KDD
- Diffusion Model：ICLR/NeurIPS 2021 → CVPR/ICCV → MM → KDD
- GAN：NeurIPS 2014 → CVPR → MM → IJCAI

延迟短（1–2 年）= 通用方法，任意领域研究者均可直接应用；延迟长（4+ 年）= 存在领域知识壁垒或特定数据/算力需求。

---

#### 模块 D：`KnowledgeFlowGraph` — 知识流动有向图

**目标**：这是本分析体系最具原创性的维度。量化 AI 各子领域之间思想迁移的方向与速度，识别哪些领域是知识输出者，哪些是知识接收者。

**方法**：
1. 对每个领域提取特征词（该领域 PMI 值显著高于全语料的词汇）
2. 计算这些特征词在其他领域论文中的出现频率，与随机期望对比
3. 若某领域特征词在目标领域中显著高于期望（PMI > 阈值），则存在从前者到后者的知识借用关系
4. 构建有向图：节点 = 领域，有向边 = 借用关系，边权重 = PMI 差值

**具体案例**：
- "retrieval"（IR 核心词）在 LLM 论文中的出现频率是否显著上升？→ IR 概念向 LLM 的输入（RAG 的知识论溯源）
- "attention"（NLP/Transformer）被 CVPR 论文采用的时间线 → NLP 方法论向视觉领域的迁移
- "embedding"（ML 词汇）在 SIGMOD 论文中的频率变化 → ML 概念向数据库系统的渗透

**输出**：`nx.DiGraph`，可使用 PyVis 导出为交互式 HTML 可视化。

### 4.2 `analysis/network_analysis.py` — 共作者网络（模块 E）

**目标**：从宏观网络结构演化视角，量化 AI 研究社区是否越来越"成为一体"。关注宏观结构规律而非单个研究者。

#### 核心类：`CoauthorNetworkAnalyzer`

**数据结构**：以作者为节点，共同署名论文数为边权重，构建加权无向图。

**主要方法**：

| 方法 | 说明 | 关键输出 |
|---|---|---|
| `build_graph(papers, year)` | 构建特定年份（或全量）的共作者图 | `nx.Graph` |
| `analyze_evolution(papers, years)` | 逐年计算网络结构指标 | `pd.DataFrame` |
| `find_bridge_researchers(G, top_n)` | 识别高 betweenness centrality 的研究者 | `List[Dict]` |
| `detect_communities(G)` | Louvain 社区发现（可选 Leiden） | `Dict` |
| `find_cross_venue_researchers(papers, venue_pairs)` | 跨会议活跃研究者 | `List[Dict]` |

**`analyze_evolution()` 逐年计算的指标**：

| 指标 | 含义 |
|---|---|
| `avg_path_length` | 平均路径长度（下降 = 社区在靠近） |
| `clustering_coefficient` | 聚类系数（上升 = 局部圈子在加密） |
| `network_density` | 网络密度（整体连通程度） |
| `num_components` | 连通分量数（下降 = 原本孤立的群体在融合） |

**桥接者识别**：高 betweenness centrality 的节点是社区间信息中介。同时在 SIGMOD+ICML 或 ACL+CVPR 发表的研究者，往往是跨社区知识迁移的实际载体，也是预测未来领域融合方向的早期信号。

**社区边界的渗透性**：Louvain 社区发现在每年结果对比，如果某个时间节点后两个本来独立的社区开始合并，往往对应一个重要新方向的成熟——这比关键词分析早 1–2 年给出信号。

## 五、单领域深度层

### 5.1 `analysis/deep_domain.py` 扩展

在现有 `DeepDomainAnalyzer` 基础上，新增以下方法并升级现有逻辑：

#### 新增：`analyze_researcher_stability(domain_papers, years) -> Dict`

计算相邻年份该领域活跃作者集合的 Jaccard 相似度（连续两年都发表至少一篇论文的作者比例）。

**输出**：

```python
{
    "year_pairs": [(2019, 2020), (2020, 2021), ...],
    "jaccard_scores": [0.23, 0.31, ...],
    "avg_stability": 0.27,
    "stability_type": "蜂拥而至型",    # Jaccard < 0.3
    "interpretation": "大量研究者短暂尝试后离开，领域缺乏稳定的知识积累核心。
                       处于爆发期，方向分散，结论难以复现。"
}
```

**稳定性类型判断标准**：
- `avg_jaccard < 0.3`：蜂拥而至型 — 爆发性热潮，外部涌入驱动
- `0.3 ≤ avg_jaccard < 0.5`：混合型 — 核心专家 + 外围涌入并存
- `avg_jaccard ≥ 0.5`：专家主导型 — 稳定的核心研究群体，知识边界清晰

#### 新增：`fit_lifecycle(yearly_counts) -> Dict`

对领域年度论文数拟合 logistic 增长模型（S 曲线），判断领域当前所处生命周期阶段。

**输出**：

```python
{
    "phase": "爆发期",
    "params": {"L": 850, "k": 0.7, "t0": 2022.3},  # 逻辑斯蒂参数
    "r_squared": 0.94,                               # 拟合优度
    "interpretation": {
        "L": "预期成熟期峰值约 850 篇/年",
        "k": "扩散速率 0.7（较快）",
        "t0": "爆发中点约 2022 年"
    },
    "phase_description": "当前处于爆发期陡坡段，论文数快速增长，研究方向开始收敛，
                          跟进门槛持续升高。"
}
```

#### 升级：`_analyze_emerging_topics`

将 `(late - early) / early > 0.5` 替换为 Mann-Kendall 非参数趋势检验（调用 `stats_utils.mann_kendall_test`）：
- 对每个候选词汇的年度频率序列运行检验
- **只报告 `p < 0.05` 的显著趋势**，消除小样本假阳性
- 附带 Sen's 斜率（每年增长的论文数）作为趋势强度指标

### 5.2 五维度深度分析（每个领域独立输出）

基于上述方法，每次单领域分析产出以下五个维度的洞见：

| 维度 | 核心问题 | 关键指标 |
|---|---|---|
| **① 创新起源地** | 哪个会议、哪一年首次出现论文集群（≥5篇/年）？ | 首次集群会议、年份、后续扩散路径 |
| **② 会议吸收速度** | 从专业会议主导到综合会议主导需要几年？ | 扩散延迟（调用模块 C） |
| **③ 词汇新陈代谢** | 哪些词在上升/下降？下降原因是"问题已解决"、"方向放弃"还是"术语更新"？ | Mann-Kendall 趋势显著性，整体论文数对比 |
| **④ 研究者稳定性** | 是专家深耕型还是蜂拥而至型爆炸增长？ | 活跃作者 Jaccard 相似度 |
| **⑤ 跨领域影响力** | 该领域的特征词是否开始在其他领域高频出现？ | PMI 传播方向（调用模块 D） |

## 八、关键架构决策

### 决策一：GPU 加速的优先级分配

本框架支持两种运行模式：

- **无 GPU 模式**：使用 sklearn TF-IDF 管道实现所有宏观分析，确保在普通笔记本上可运行
- **GPU 加速模式**：使用 BERTopic + SPECTER2 进行语义嵌入和自动主题发现，需要 NVIDIA GPU（推荐 8GB+ 显存）

用户本地有 RTX 3070 Ti (8GB)，足以运行：
- SPECTER2：~4GB 显存，用于论文语义嵌入
- BERTopic + BERT：~6GB 显存，用于自动发现子方向
- 分批处理策略：避免一次性加载全部 25 万篇论文，控制每批大小

### 决策二：`NgramExtractor` 独立封装

N-gram 提取器作为独立模块，其他模块通过组合而非继承使用，避免循环依赖。`ecosystem.py`、`deep_domain.py`、`trend_analysis.py` 等均可按需引入 `NgramExtractor`。

### 决策三：pandas DataFrame 作为分析结果容器

宏观分析的结果（词汇时间轴、相似度矩阵、网络演化）使用 `pd.DataFrame` 存储，便于：
- CSV 直接导出，可在 Excel/Python 中进一步处理
- 链式 `.groupby()`、`.pivot()` 操作
- 未来接入 matplotlib/seaborn/Plotly 可视化

### 决策四：统一缓存策略

所有计算密集操作（N-gram 训练、TF-IDF 矩阵、网络构建）使用基于内容哈希的 pickle 缓存：
- 缓存路径：`analysis/cache/{模块名}_{参数哈希16位}.pkl`
- 哈希基于输入数据内容，数据变化时自动失效
- 25 万篇论文全量分析一次 N-gram 训练约需 2–5 分钟，缓存后后续运行 < 5 秒

### 决策五：Mann-Kendall 取代朴素增长率

对所有"趋势判断"统一使用 Mann-Kendall 非参数检验：
- 无需正态分布假设，适合小样本的年度频率数据
- 输出 p 值，只报告 `p < 0.05` 的显著趋势
- Sen's 斜率提供每年绝对增长量，便于横向对比

## 九、可回答的 20 个具体研究问题

### 全领域问题（宏观生态层）

1. AI 研究中哪些技术词汇**首次出现即成主流**（扩散速度 < 1 年）？哪些**长期保持小众**（5 年后仍未在第二个会议集中出现）？
2. 哪两个会议之间的**研究重心融合最快**（相似度矩阵 2015–2018 与 2023–2025 差值最大）？
3. 哪些研究者是跨会议的**知识传播节点**（betweenness centrality 最高），他们主要横跨哪两个领域？
4. 整个 AI 研究社区的**平均路径长度**是否在缩短？从 2015 到 2025 缩短了多少？
5. 哪些领域是**知识输出者**（特征词被大量借用），哪些是**知识接收者**？知识流动方向是否随时间变化？
6. **Transformer** 在各会议的扩散延迟分别是多少？与 **GAN** 的历史扩散速度相比如何？
7. 2015–2025 年间，哪个年份是 AI 研究社区**合作密度**增长最快的年份？对应什么历史事件？

### 单领域问题（深度分析层）

8. **联邦学习**是否已过峰值？S 曲线拟合显示处于哪个阶段？研究者稳定性如何？哪个子方向（Privacy / Efficiency / Non-IID）正在衰退？
9. **Graph Neural Networks** 的理论类论文和应用类论文的比例如何变化？哪个应用领域的采用率最高？GNN 从 NeurIPS 扩散到推荐系统（SIGIR/KDD）的延迟是多少年？
10. **Diffusion Models** 从 ICLR/NeurIPS 扩散到 MM（多媒体）的延迟是多少年？与 GAN 的历史扩散速度相比如何？
11. **AI Agent** 研究是否分裂为"NLP 学派"（语言交互 → 起源 ACL）和"RL 学派"（规划决策 → 起源 ICML）两个平行传统？两个学派的词汇有何差异？
12. **推荐系统**领域在 LLM 时代的词汇新陈代谢——"collaborative filtering"、"matrix factorization" 何时开始显著下降（Mann-Kendall p < 0.05）？"generative recommendation" 何时开始显著上升？
13. **大语言模型**研究的研究者稳定性 Jaccard 系数是多少？是专家深耕型还是蜂拥而至型？这能否解释该领域的"内卷化"现象？
14. **对比学习**（contrastive learning）的词汇生命周期如何？它的 S 曲线是否已进入平台期？
15. 哪些 CCF-A 会议是 **RAG（检索增强生成）** 研究的主战场？RAG 起源于哪个社区？"信息检索"社区（SIGIR）的参与度如何演变？

### 交叉分析问题

16. 在**知识流动有向图**中，哪个领域在 2015–2025 期间累计贡献了最多"被借用词汇"？
17. 对于**同一篇论文同时属于多个领域**（软归属得分均 > 0.3）的论文，其出现频率在 2020–2025 年间如何变化？这是否预示着领域边界正在消融？
18. **桥接 ACL + CVPR** 的研究者（多模态方向先驱）最早出现在哪一年？他们的 betweenness centrality 是否在 2021–2022 年（CLIP 爆发期）显著上升？
19. 对比**联邦学习**（专业会议主导，ICML/ICLR 起源）与**推荐系统**（应用会议主导，KDD/SIGIR 起源）两个领域，哪个领域在 LLM 时代受到的冲击更大（词汇新陈代谢更剧烈）？
20. 2022–2025 年间，SIGMOD/ICDE 论文中 "LLM"、"embedding"、"vector" 等机器学习词汇的出现频率是否呈现显著上升趋势（Mann-Kendall p < 0.05）？这能否量化"AI for Databases"的渗透速度？

## 十、快速上手指南

### 安装依赖

```bash
# 核心依赖（必装）
pip install -r requirements.txt

# 可选：高级主题建模（需 GPU 推荐）
pip install bertopic keybert

# 可选：交互式可视化
pip install pyvis

# GPU 加速（推荐 RTX 3070 Ti 或更高）
pip install bertopic keybert sentence-transformers torch
```

### 基本使用流程

```bash
# 1. 查看可用数据
python -m analysis list
python -m analysis stats

# 2. 运行词汇时间轴（方法论基础验证）
python -m analysis timeline --years 2015,2025 --top-n 50 --output ./output/analysis/

# 3. 宏观生态全量分析
python -m analysis ecosystem --years 2015,2025 --output ./output/analysis/ecosystem/

# 4. 共作者网络分析
python -m analysis network --years 2015,2025 --output ./output/analysis/network/

# 5. 单领域深度分析
python -m analysis deep "Large Language Models" --years 2020,2025 --output ./output/analysis/
python -m analysis deep "Federated Learning" --years 2018,2025 --output ./output/analysis/
python -m analysis deep "Graph Neural Networks" --years 2018,2025 --output ./output/analysis/
```
